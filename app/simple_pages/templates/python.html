PYTHON / FLASK

Flask is a micro web framework written in Python. It is classified as a microframework because it does not require particular tools or libraries. It has no database abstraction layer, form validation, or any other components where pre-existing third-party libraries provide common functions. It can also be considered a back-end framework, which means that it provides the technologies, tools, and modules that can be used to build the actual functionalities of the web app rather than the design or look of it.

Testing. Automating and Validating.

PyTest is a testing framework that allows users to write test codes using Python programming language. It helps you to write simple and scalable test cases for databases, APIs, or UI. PyTest is mainly used for writing tests for APIs. It helps to write tests from simple unit tests to complex functional tests.

  • Pytest allows you to set requirements that are either validated or failed during the test run.

    Purpose. What do these files do?.

    The combinations of all these files create the application itself which will get deployed to the cloud and browser. Here is a brief description of those files:

    • Project Root Folder: This is the main root folder where all the application files will be stored.
    • Workflows: This folder contains the .yml files (dev, prod, and heroku). Actions, jobs, steps, logins, passwords, uses, etc.. are set with these files.
    • Readme file: Contains instructions, directions and comments made by the author.
    • Simple_pages/Templates Folder: Contains all hmtl files
    • Static Folder: Contains the css file, images files, and js file.
    • __init_.py: The main folder is our Python directory that we want to treat as if it were a Python package.
      To be treated as such, we have to include an __init__.py file that relays this information to the Python interpreter.
    • Tests Folder: Contains the .py tests syntax used to run the PyTest on requirements.
    • .pylintrc: Helps execute semantics and "grammar" tests on code.
    • Docker-compose.yml: This (orchestration file) is the registration file, it ties everything together. Gives instructions to start up and run programing language, and open up the network ie 8080. Nginx is the web server, this is the networking connecting to the outside world.
    • Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image. Using docker build users can create an automated build that executes several command-line instructions in succession.
